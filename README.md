## 🏗️ Architecture & System Design

I think in **services, contracts, and failure modes**. Some of the patterns I reach for and ship:

- **Async-first APIs** — FastAPI + SQLAlchemy 2.0 async + asyncpg for high-throughput backends
- **Event-driven workflows** — RabbitMQ + Celery for background jobs, retries with exponential backoff, and dead-letter queues
- **Domain-driven modeling** — clear separation between domain, application, and infrastructure layers
- **Caching strategies** — Redis for read-through caching, rate limiting, distributed locks, and session storage
- **API design** — RESTful + OpenAPI-first; role-filtered specs (admin / provider / public) for large surfaces (180+ endpoints)
- **Database engineering** — normalized schemas, ENUM types, triggers (e.g., `avg_rating` recalculation, balance protection), strategic indexing, and Alembic migrations as code
- **Observability** — structured logging, health checks, request tracing, and alerting hooks built in from day one
